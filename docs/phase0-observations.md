# Phase 0 Observations

## 我們做出了什麼？

- 建立一個可從首頁進入的災害資訊整理工作台，讓下一位協作者能查看原始資訊、建立整理草稿、編輯判斷與標示是否可直接變成任務。
- 讓工作台顯示「人類修正」與「不能直接變成任務」兩種關鍵提醒，避免把未確認資訊當成正式任務。
- 把三筆較有脈絡的記錄整理成候選結果，並保留為需人工確認的狀態。

## 哪些資料讓 agent 很容易整理？

- M-009 與 M-010 的現場志工更新內容比較具體，包含集合點、物資剩餘量與盤點時間，因此比較容易被整理成候選的地點狀態更新。
- M-003 也有明確提到資源需求的變化，雖然仍不夠安全，但比一般社群貼文更容易被辨認成資源狀態訊息。

## 哪些資料讓 agent 亂補或誤判？

- M-001 與 M-004 這類社群貼文看起來像請求，但缺少精確地址與確認者，任何直接轉成任務的判斷都會有風險。
- M-005 的截圖內容沒有說明來源、時間與是否為官方公告，因此不能被當成正式公告。
- M-012 的來電資訊涉及親友與位置不明，若直接轉成任務，會把不確定資訊當成事實。

## 哪些資訊不能直接變成任務？

- M-001：缺少精確地點與確認人，不能直接派工。
- M-011：涉及長者與隱私位置，不能直接公開或直接建立任務。
- M-012：來電者無法確認親友當下位置，不能直接建立任務。

## 哪些資訊的操作者不是當事人？

- M-001、M-004、M-005、M-007 這些都來自社群或轉述，不一定是當事人本身。
- M-011 是志工代為轉述，並非長者本人直接回報。
- M-012 是外地家屬來電，不是當事人現場狀態。

## 我們剛剛偷偷假設了什麼？

- 我們假設「看起來像求助」就可以先做候選整理，但實際上這仍須由人類核對。
- 我們假設現場志工更新比較可靠，但沒有官方同步，仍然不能假設為已確認事實。

## 如果要讓這個工具變可靠，下一步需要什麼？

- 需要欄位來記錄來源確認者、更新時間與確認方式，才能更清楚哪些資訊可以被信任。
- 需要把「是否允許公開地址」「是否已取得當事人同意」等欄位納入草稿。
- 需要在下一階段把多筆資訊的對比與版本差異也呈現出來。

## 我們需要更清楚資料格式的地方

- 原始資料目前沒有明確的「確認者」「確認方式」「地點精度」欄位。
- 目前沒有標示某筆資訊是「來源轉述」還是「直接當事人提供」。

## 我們需要人工確認的地方

- 所有含有社群轉述、電話轉述或現場口述但未見官方同步的資訊，都需要人工確認。
- 任何涉及個人位置、隱私或家屬代為轉述的資訊，都應由人員先確認再進一步處理。
